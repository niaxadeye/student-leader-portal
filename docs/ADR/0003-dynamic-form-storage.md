# ADR 0003. Хранение динамических форм

## Статус
Принято.

## Контекст
Форму испытания нельзя хардкодить: администратор конструирует поля, схема меняется,
но старые ответы должны оставаться валидными (§11, §12).

## Решение
- Статусы и связи — обычные таблицы (`submissions`, `challenge_fields`).
- Динамические ответы — `JSONB` (`answers_json`), валидация по серверной схеме.
- Версионирование: `challenge_schema_versions` хранит snapshot схемы; несовместимые
  изменения требуют подтверждения, поля — soft delete.
- Ревизии (`submission_revisions`) immutable: snapshot схемы + ответов + файлов + checksum.

## Последствия
- Гибкость без миграций под каждое поле.
- Ответы прошлых ревизий читаемы по своей версии схемы.
- Валидацию JSONB держим на backend; индексируем только нужные пути при необходимости.
