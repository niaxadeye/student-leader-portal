# ADR 0005. Transactional outbox

## Статус
Принято.

## Контекст
Отправка формы должна порождать Telegram-уведомление, но сбой Telegram не должен
откатывать успешную отправку, а событие не должно теряться или дублироваться (§15).

## Решение
- Отправка формы и запись `outbox_events` — в одной транзакции.
- Worker выбирает `PENDING` с `FOR UPDATE SKIP LOCKED`, отправляет, пишет результат.
- Retry с exponential backoff; после лимита попыток статус `DEAD`.
- Доставки логируются в `notification_deliveries`, идемпотентность по событию.

## Последствия
- Гарантия at-least-once доставки без распределённых транзакций.
- Сбой внешнего сервиса не влияет на основную бизнес-операцию.
- Тот же паттерн переиспользуется для будущих side effects (email, экспорт).
