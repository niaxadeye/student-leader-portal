# ADR 0001. Модульный монолит

## Статус
Принято.

## Контекст
Нужен production-ready кабинет с конкурсным модулем и заделом под будущие модули
(участник, лекции, баллы, мерч, прогнозы). Команда небольшая, сервер один.

## Решение
Один Go-процесс API + отдельный worker. Внутри — модули (`internal/modules/*`)
с чёткими границами и интерфейсами на внешних краях (storage, telegram, clock, id, tx).
Микросервисы не вводим.

## Последствия
- Простой деплой и отладка, одна транзакционная БД.
- Будущие модули добавляются как новые пакеты без переписывания ядра.
- Границы модулей соблюдаем дисциплиной, а не сетью — при росте можно выделить сервисы.
