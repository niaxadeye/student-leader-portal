# ADR 0004. Файловое хранилище

## Статус
Принято.

## Контекст
Конкурсанты загружают большие файлы (презентации, видео). PostgreSQL для бинарных
данных не подходит; bucket не должен быть публичным (§13).

## Решение
- Объекты в S3-совместимом хранилище (production) / MinIO (development).
- В PostgreSQL — только метаданные (`files`, `submission_files`).
- Загрузка через presigned URL: init → presigned PUT → complete → серверная проверка →
  статус READY. Предусмотрен multipart.
- Скачивание через backend-проверку прав (не по знанию URL), presigned с коротким TTL.
- Удаление — soft delete, физическое — worker; файлы прошлых ревизий сохраняются.

## Последствия
- API не гоняет тело файла через себя — масштабируется.
- Требуется интерфейс `Storage` и `FileScanner` (антивирус) на границе.
- Object key генерируется сервером — защита от path traversal.
